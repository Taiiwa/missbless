Git多人项目

线上

  • master - 主分支，提交已经通过测试的代码
  • develop - 测试环境，服务器环境
  • 管理分支 - 可以新建分支，为程序员创建个人分支

本地

  • 每个人在本地创建个人分支

      mkdir xx

  git init 
    
  git clone "master的ssh"
    
  # 进行代码更新操作


  # 创建分支
  git branch develop origin/develop

  git branch xx origin/xx

  # 查看分支
  git branch

  # 切换节点
  git checkout xx

  # 整合到测试环境
  git add -A
  git commit -m ""
  git push origin xx

  # 切换
  git checkout develop

  # 整合
  git merge xx

  • 向分支上推送代码：

      git clone SSH

  cd 项目名

  git branch 分支名 origin/分支名

  git checkout 分支名

  git add -A

  git commit -m ""

  git push -u origin 分支名

  • 冲突

      git status

  # 显示冲突文件名

  git diff 文件名

  # pull报错：
  git pull origin develop
  fatal: Exiting because of an unresolved conflict.

  git reset --hard FETCH_HEAD

  # 重新拉取
  git pull origin develop

  git merge 分支

  git push origin develop
